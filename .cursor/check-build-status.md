# 检查构建状态 - 使用 GitHub MCP Server

由于 GitHub API 需要认证，请在 Cursor 中使用 GitHub MCP Server 的自然语言接口：

## 📋 检查构建状态

在 Cursor 聊天中直接说：

```
"列出 Build WinUnlock 工作流程的最新运行"
```

或者：

```
"列出分支为 dev 的工作流程运行"
```

## 📊 获取运行详情

```
"获取最新工作流程运行的详情"
```

或者指定运行 ID：

```
"获取工作流程运行 #19604278709 的详情"
```

## 📥 下载错误日志（如果失败）

```
"下载最新失败的工作流程运行的日志"
```

或者：

```
"下载工作流程运行 #19604278709 的日志"
```

## 🔄 完全自动化检查

```
"检查最新构建状态，如果失败就获取错误日志并自动修复"
```

---

**注意**：这些命令需要通过 Cursor 的 GitHub MCP Server 自然语言接口调用。
